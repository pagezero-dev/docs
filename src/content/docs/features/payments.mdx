Payment integration powered by **[Polar.sh](https://polar.sh/)**. Sell digital products, subscriptions, or one-time purchases. Payments are handled entirely by Polar—no Stripe setup, webhook complexity, or compliance headaches.

**Key features**

- One-time payments and subscriptions
- Automatic role assignment on purchase
- Email notifications for access changes
- Ready-to-use checkout button and pricing components
- Webhook handling for payment events

## Setup

1. Create an account at [polar.sh](https://polar.sh/)
2. Create your products in the Polar dashboard
3. Generate checkout links for each product
4. Create a webhook endpoint pointing to `/payments/webhook`
5. Add `POLAR_WEBHOOK_SECRET` as a secret in Cloudflare (see [Cloudflare setup](/getting-started/deployment#cloudflare-setup))

Configure your products in `packages/config/index.ts`:

```typescript
payments: {
  products: {
    premium: {
      name: "Premium",
      userRoleToGrant: "premium",
      polarProductId: {
        preview: "your-preview-product-id",
        production: "your-production-product-id",
      },
      checkoutLink: {
        preview: "https://sandbox-api.polar.sh/v1/checkout-links/.../redirect",
        production: "https://api.polar.sh/v1/checkout-links/.../redirect",
      },
    },
  },
}
```

Each product maps to a user role. When a customer completes payment, they're automatically granted the corresponding role.

## API

| Function | Module | Purpose |
|----------|--------|---------|
| `onPaymentSuccess(event, context)` | `@/payments/handlers.server` | Handle successful payments |
| `onPaymentRevoked(event, context)` | `@/payments/handlers.server` | Handle refunds and cancellations |

## Components

### CheckoutButton

A styled button that links to the Polar checkout page. Automatically pre-fills the customer's email if they're logged in.

```tsx
import { CheckoutButton } from "@/payments/components/checkout-button"

<CheckoutButton productId="premium">
  Get Premium Access
</CheckoutButton>
```

The `productId` must match a key in your `payments.products` config.

## How It Works

**Purchase flow:**

1. User clicks a `CheckoutButton`
2. User is redirected to Polar's hosted checkout
3. After payment, Polar sends a webhook to `/payments/webhook`
4. The webhook handler grants the user role and sends a confirmation email
5. User is redirected to `/payments/success`

**Webhook events handled:**

| Event | Action |
|-------|--------|
| `order.paid` | Grant role, send access granted email |
| `subscription.active` | Grant role, send access granted email |
| `order.refunded` | Revoke role, send access revoked email |
| `subscription.revoked` | Revoke role, send access revoked email |

## Permissions Integration

Payments integrate with the [permissions system](/features/permissions). When configuring a product, set `userRoleToGrant` to the role you want customers to receive:

```typescript
products: {
  elite: {
    name: "Elite",
    userRoleToGrant: "elite", // Grants the "elite" role on purchase
    // ...
  },
}
```

The role is automatically revoked on refund or subscription cancellation.

## Routes

| Route | Purpose |
|-------|---------|
| `/payments/webhook` | Receives webhook events from Polar |
| `/payments/success` | Success page shown after checkout |

## Testing

In development, use Polar's sandbox environment:

1. Set `preview` product IDs and checkout links from Polar sandbox
2. Use test card numbers provided by Polar
3. Verify webhooks are received and roles are assigned correctly

For production, update the `production` values in your config with live product IDs and checkout links.
